Topaz.Display 


 An abstraction for the environment's means of 
 displaying rendered information.

 In most contexts / systems, the display is a window within the 
 desktop environment. On embedded systems, this may be the entire
 screen display.

 Because of the flexibility / variation in what a display can be
 some backends may choose to implement features differently
 or omit features altogether where appropriate.

Topaz.Display.VIEW_POLICY 

  Controls how the Display displays Rendered data.
NONE The Display will show the attached Framebuffer's contents with no transformation The visual is stretched over the width and height of the display.
MATCH_SIZE The Display will resize the attached Framebuffer to match the windows dimensions whenever the window is resized. This is the default.

(Enumerator)

Topaz.Display.PARAMETER 

 On most systems, displays have a variety of features that can be 
 controlled. This provides a generic interface into those features,
 where a request can be posted to change a parameter via 
 topaz_display_set_parameter().
X Controls the X position of the display. Usually, this is relative to whatever environment the display exists in. For example, in a desktop environment, this could be an offset from the DE's origin. If the display does not support moving, no action is taken.
Y Controls the Y position of the display. Usually, this is relative to whatever environment the display exists in. For example, in a desktop environment, this could be an offset from the DE's origin. If the display does not support moving, no action is taken.
WIDTH The width of the display.
HEIGHT The height of the display.
SHOW Controls whether the display is shown.
FULLSCREEN Set the display into a fullscreen context. If fullscreen is not supported, no action is taken.
LOCK_CLIENT_RESIZE Controls prevention of resizing on the user's side. For example, in a desktop environment, this would disable the feature of resizing the window.
LOCK_CLIENT_POSITION Attempts to prevent moving on the user's side.
VIEW_POLICY Controls how the Renderer's information is displayed. The default policy is "MatchSize" See ViewPolicy for more information.
INPUT_FOCUS Returns whether the display has user input focus. On display implementations where this doesnt apply, i.e. where there is only one logical display available,, this will always return true.
ACTIVE Determines whether the display is active. Active displays are regularly updated by topaz, which means input is polled, their root is run, and data is shown on it. Displays are active by default

(Function)

captureMainFramebuffer(

Topaz.Image capture

) 

 Gets the contents of the current framebuffer and 
 puts the data into an image asset frame. 
 The input image is cleared, resized to match the 
 framebuffer size, and given one frame which matches the 
 the framebuffer data.

 NOTE: in general, this operation is considered "slow" as 
 it requires a lot of communication between the backend 
 renderer and engine, which are most likely on different 
 devices. Use sparringly!
Arguments:

capture 
 The image to populate.
